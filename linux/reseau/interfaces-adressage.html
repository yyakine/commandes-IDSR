    Interfaces et adressage

    Sous Linux, la gestion réseau passe par deux couches : les commandes ip pour la configuration temporaire (perdue au reboot), et les fichiers de configuration persistants qui diffèrent selon la distribution.

    Inspecter les interfaces

    ip link show                  # Lister toutes les interfaces et leur état (UP/DOWN)
ip addr show                  # Lister les interfaces avec leurs adresses IP
ip addr show eth0             # Détails d'une interface spécifique
ip -brief addr show           # Vue compacte : interface, état, IP
ip -4 addr show               # IPv4 uniquement
ip -6 addr show               # IPv6 uniquement

    Configuration temporaire (ip)

    Ces commandes s'appliquent immédiatement mais sont perdues au redémarrage.

    # Adresse IP
ip addr add 192.168.1.10/24 dev eth0      # Ajouter une adresse IP
ip addr del 192.168.1.10/24 dev eth0      # Supprimer une adresse IP

# État de l'interface
ip link set eth0 up                        # Activer une interface
ip link set eth0 down                      # Désactiver une interface

# Adresse MAC temporaire
ip link set eth0 address 00:11:22:33:44:55

    Configuration persistante

    Debian utilise Netplan (Ubuntu 18.04+).

    Netplan — fichier YAML dans /etc/netplan/ :

    /etc/netplan/01-netcfg.yaml — IP statique
    network:
  version: 2
  ethernets:
    eth0:
      addresses:
        - 192.168.1.10/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 1.1.1.1]
      dhcp4: false
    /etc/netplan/01-netcfg.yaml — DHCP
    network:
  version: 2
  ethernets:
    eth0:
      dhcp4: true

    netplan apply               # Appliquer la configuration Netplan
netplan try                 # Tester avec rollback automatique si pas confirmé

    Red Hat utilise NetworkManager via nmcli. nmcli:

    IP statique avec nmcli
    nmcli con mod eth0 ipv4.addresses 192.168.1.10/24
nmcli con mod eth0 ipv4.gateway 192.168.1.1
nmcli con mod eth0 ipv4.dns "8.8.8.8 1.1.1.1"
nmcli con mod eth0 ipv4.method manual
nmcli con up eth0
    DHCP avec nmcli
    nmcli con mod eth0 ipv4.method auto
nmcli con up eth0
    Créer une nouvelle connexion
    nmcli con add type ethernet ifname eth0 con-name eth0 \
  ipv4.addresses 192.168.1.10/24 \
  ipv4.gateway 192.168.1.1 \
  ipv4.dns "8.8.8.8" \
  ipv4.method manual

    nmcli con reload            # Recharger les fichiers de configuration
nmcli con up eth0           # Appliquer sur l'interface

    Alias et adresses multiples

    # Ajouter une deuxième adresse sur la même interface
ip addr add 10.0.0.1/24 dev eth0

# Vérifier
ip addr show eth0

    Plusieurs adresses sur une interface

    Contrairement aux sous-interfaces (eth0:0 de l'ancienne méthode), ip addr add permet d'assigner plusieurs adresses à la même interface directement, sans créer d'alias virtuel.
